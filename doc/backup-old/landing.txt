 {/*<H1>Why ?</H1>*/}

{/*<p>*/}
{/*    Forms are one of the most essential yet challenging aspects of*/}
{/*    web development. Despite their importance, building robust and*/}
{/*    maintainable forms often leads to duplicated validation logic,*/}
{/*    fragile data handling, and tightly-coupled UI constraints and*/}
{/*    challenges that become increasingly complex as application*/}
{/*    requirements evolve. We built Kofono to solve the following*/}
{/*    problems:*/}
{/*</p>*/}
{/*<ul class="list-disc pl-5 flex flex-col gap-4">*/}
{/*    <li>*/}
{/*        <StrongAccent>Schema base</StrongAccent> Kofono schema base,*/}
{/*        which provides a an easy way and type system to describe the*/}
{/*        structure of your form data.*/}
{/*    </li>*/}
{/*    <li>*/}
{/*        <StrongAccent>Complex forms</StrongAccent> : Kofono allows*/}
{/*        you to define complex forms with a simple schema. It*/}
{/*        provides a powerful and extensive API to tackle almost any*/}
{/*        use case.*/}
{/*    </li>*/}
{/*    <li>*/}
{/*        <StrongAccent>Validations</StrongAccent> : Validation logic*/}
{/*        lives in multiple places (client, server, DB) with no*/}
{/*        coordination. Kofono provides a powerful validation engine*/}
{/*        to validate your form data.*/}
{/*    </li>*/}
{/*    <li>*/}
{/*        <StrongAccent>Type safety</StrongAccent> : Kofono is built*/}
{/*        with TypeScript and provides full type safety for your form*/}
{/*        schema.*/}
{/*    </li>*/}
{/*    <li>*/}
{/*        <StrongAccent>Headless</StrongAccent> : Kofono core is not*/}
{/*        tied to any frameworks or ui, so it can be used on the*/}
{/*        frontend and the backend. No duplicate logic or validations.*/}
{/*        You can reuse your form schema on the backend to validate*/}
{/*        incoming data. This also mean you can unit test your form*/}
{/*        schema.*/}
{/*    </li>*/}
{/*    <li>*/}
{/*        <StrongAccent>Extensibility</StrongAccent> : Kofono is*/}
{/*        designed to be easily extendable. You can create your own*/}
{/*        validators, schema plugins, custom UI components/renderer,*/}
{/*        and more.*/}
{/*    </li>*/}
{/*</ul>*/}
